# migrate-to-csv

**Migration utilities to convert existing JSON data to CSV format.**

## Purpose

The migration scripts convert existing JSON-based pipeline data to the new CSV format defined in `config/pipeline_steps.json`. These scripts preserve all downloaded and analyzed data that would be expensive or impossible to regenerate (API calls, LLM analysis).

**What gets migrated:**
- Process 1: Exchange Rates (from OpenExchangeRates API)
- Process 3: News Articles (from NewsAPI and RSS feeds)
- Process 4: Time Horizon Analysis (LLM-analyzed data)
- Process 5: Sentiment Signals (LLM-generated signals)

**Why these processes:** They contain source data that can't be regenerated without cost or API access. Other processes (2, 6, 7, 8, 9) derive their data from these and can be regenerated by rerunning the pipeline.


## Available Scripts

### Master Script: `migrate_all_to_csv.py`

Runs all migrations in sequence with consolidated reporting.

**Usage:**
```bash
# Dry run (preview)
python3 scripts/utilities/migrate_all_to_csv.py --dry-run

# Migrate everything
python3 scripts/utilities/migrate_all_to_csv.py

# Migrate specific process
python3 scripts/utilities/migrate_all_to_csv.py --process 1

# Migrate specific date
python3 scripts/utilities/migrate_all_to_csv.py --date 2026-02-24
```

**Options:**
- `--dry-run`: Preview changes without modifying files
- `--date YYYY-MM-DD`: Migrate only specific date
- `--process {1,3,4,5}`: Migrate only specific process

**Features:**
- Interactive prompts if migration fails (continue or abort)
- Consolidated summary report
- Exit with error code if any migration fails

## Migration Strategy

### What Gets Migrated

These processes contain **source data** that can't be regenerated:

| Process | Data Type | Why Preserve |
|---------|-----------|--------------|
| 1 | Exchange Rates | API calls (OpenExchangeRates) |
| 3 | News Articles | API calls (NewsAPI, RSS feeds) |
| 4 | Time Horizons | LLM analysis (expensive) |
| 5 | Sentiment Signals | LLM generation (expensive) |

### What Doesn't Get Migrated

These processes **derive** their data and can be regenerated:

| Process | Can Regenerate By |
|---------|-------------------|
| 2 | Running Process 1 → 2 |
| 6 | Running Processes 2,4,5 → 6 |
| 7 | Running Process 6 → 7 |
| 8 | Running Process 7 → 8 |
| 9 | Running Processes 1,8 → 9 |

**To regenerate after migration:**
```bash
# Rerun entire pipeline for a date
python3 scripts/utilities/run-system.py --date 2026-02-24

# Or rerun specific processes
python3 scripts/utilities/run-system.py --date 2026-02-24 --process-ids 2 6 7 8 9
```

## Error Handling

### Missing Directories

If a data directory doesn't exist, the script will report an error:

```
Error: Prices directory not found: /workspace/group/fx-portfolio/data/prices
```

### Invalid JSON

If a JSON file is malformed:

```
Error loading data/prices/fx-rates-2026-02-24.json: Expecting value: line 1 column 1 (char 0)
```

### Empty Files

If a JSON file has no data:

```
Warning: No data found in /workspace/group/fx-portfolio/data/prices/fx-rates-2026-02-24.json
```

### Interactive Failure Handling

When running `migrate_all_to_csv.py` (not dry-run), if a process fails:

```
⚠ Warning: Process 3 migration failed
Continue with remaining migrations? (y/n):
```

## After Migration

Once migration is complete:

1. **Verify CSV files:**
   ```bash
   # Check that CSV files exist
   ls -lh data/prices/*.csv
   ls -lh data/news/*.csv
   ls -lh data/article-analysis/*.csv
   ls -lh data/signals/*.csv
   ```

2. **Regenerate derived data:**
   ```bash
   # Rerun processes 2, 6, 7, 8, 9 for recent dates
   python3 scripts/utilities/run-system.py --dates 2026-02-24 2026-02-25 --process-ids 2 6 7 8 9
   ```

3. **Update pipeline scripts** to read/write CSV instead of JSON (Phase 2)

4. **Remove old JSON files** (once verified CSV migration successful):
   ```bash
   # Remove old JSON files (ONLY after verification!)
   rm data/prices/fx-rates-*.json
   rm -r data/news/EUR/ data/news/USD/ data/news/GBP/ # ... etc
   rm data/article-analysis/*.json
   rm -r data/signals/EUR/ data/signals/USD/ # ... etc
   ```

## Troubleshooting

### "No files found to migrate"

- Check that JSON files exist in expected locations
- Verify correct directory structure
- Check file naming conventions

### "Invalid date format"

- Ensure filenames use YYYY-MM-DD format
- Check for typos in filename dates

### CSV file has fewer rows than expected

- Check JSON file for empty `articles`, `analyses`, or `signals` arrays
- Verify JSON structure matches expected format
- Look for errors in script output

### Migration fails partway through

- Use `--dry-run` to preview before running
- Migrate one date at a time to isolate issues
- Check disk space availability
